java笔试面试复习准备

2019-03-16
  • Java

1、脚本语言（解释型语言）：一条一条语句通过解释器解释执行，即边解释边执行

2、编译型语言：整体编译成二进制代码文件后，链接，执行

3、android 的底层核心是linux

4、

JVM ：英文名称（Java Virtual Machine） Java 虚拟机。执行字节码文件，jvm 是 Java 能够跨平台的核心

JRE ：英文名称（Java Runtime Environment）Java 运行时环境。包含jvm 的标准实现和 Java 的一些基本类库。

JDK ：英文名称（Java Development Kit），Java 开发工具包。

JDK>JRE>JVM。

5、编译器将java源文件编译成字节码文件，jvm使用类加载器将字节码文件加载到内存，

再使用字节码验证器验证其合法性

6、java基本数据类型（八种）：boolean，byte，short，int，long，float，double，char

7、switch case语句中break注意：若不存在break，则找到匹配的case后还会执行后面case的语句，不论后面的case是否匹配，如果未找到匹配的case直接进入default，case后的必须是常量

运算符的优先级问题

8、string类型比较地址是否相等：==

比较内容是否相同：equals()

9、数组创建后，数值型默认赋值为0，字符型默认为‘\u0000’，布尔型默认为false

10、 ⑴强引用（StrongReference）
强引用是使用最普遍的引用。如果一个对象具有强引用，那垃圾回收器绝不会回收它。当内存空间不足，Java虚拟机宁愿抛出OutOfMemoryError错误，使程序异常终止，也不会靠随意回收具有强引用的对象来解决内存不足的问题。 ps：强引用其实也就是我们平时A a = new A()这个意思。

⑵软引用（SoftReference）
如果一个对象只具有软引用，则内存空间足够，垃圾回收器就不会回收它；如果内存空间不足了，就会回收这些对象的内存。只要垃圾回收器没有回收它，该对象就可以被程序使用。软引用可用来实现内存敏感的高速缓存（下文给出示例）。

⑶弱引用（WeakReference）
弱引用与软引用的区别在于：只具有弱引用的对象拥有更短暂的生命周期。在垃圾回收器线程扫描它所管辖的内存区域的过程中，一旦发现了只具有弱引用的对象，不管当前内存空间足够与否，都会回收它的内存。不过，由于垃圾回收器是一个优先级很低的线程，因此不一定会很快发现那些只具有弱引用的对象。

⑷虚引用（PhantomReference）
“虚引用”顾名思义，就是形同虚设，与其他几种引用都不同，虚引用并不会决定对象的生命周期。如果一个对象仅持有虚引用，那么它就和没有任何引用一样，在任何时候都可能被垃圾回收器回收。
虚引用主要用来跟踪对象被垃圾回收器回收的活动。

11、java中内存管理

JVM的内存可分为3个区：堆(heap)、栈(stack)和方法区(method，也叫静态区)：

堆区: （用于动态内存分配）
jvm只有一个堆区(heap)，且被所有线程共享，堆中只存放对象本身和数组本身；

栈区:
每个线程包含一个栈区，不共享，保存基本数据类型和对象的引用；
栈分为3个部分：基本类型变量区、执行环境上下文、操作指令区(存放操作指令)；

方法区（静态区）:
所有线程共享，方法区中包含的都是在整个程序中永远唯一的元素，是类的原始代码存放处，存放static变量。

JVM的进程中，每个线程都会拥有一个方法调用栈，用来跟踪线程运行中一系列的方法调用过程

12、文件中只能有一个公共类，且与文件名同名

13、构造方法：无返回值，可以重载，必须是public，在new时调用构造方法

14、引用类型中的变量会被默认赋值，数值型为0，布尔型为false，对象为null，字符型为’\u0000’

而方法中的局部变量并没有赋予默认值

15、可见性修饰符

  • public 所有可见
  • private 仅对本类可见
  • default 本包可见(没有修饰符)
  • protected 本包和其他包的子类可见

16、this的用法，引用自身对象，在构造方法中引用其他构造方法

17、string：一旦创建，内容不可改变

​ String：适用于少量的字符串操作的情况

　　StringBuilder：对对象直接更改，线程不安全

　　StringBuffer：对对象直接更改，线程安全

在这方面运行速度快慢为：StringBuilder > StringBuffer > String

18、父类中的构造方法不会被子类继承，只能使用super在子类的构造方法中调用，若不屑super()，子类构造方法也会默认先调用父类的无参构造方法

19、可见的方法，和非静态方法才能被重写

20、多态：父类型的变量可以引用子类型的对象

动态绑定：引用调用哪个变量由对象的实际类型决定，而不是声明类型

21、ArrayList和vector :使用数组实现，查找方便，但插入和删除需要移动大量元素，容量不够时，还需要扩容

LinkList：使用循环双向链表实现，插入和删除比较方便

22、final修饰的类和方法不能被重写和扩展，子类只能保持或者扩大父类中方法的可访问性

23、异常的根类：Throwable

24、

抽象类：1、可以不包含抽象方法

2、子类继承需全部实现抽象类中的抽象方法，否则必须被声明为抽象类

3、抽象方法必须被声明为public或者时protected

接口：1、接口中的所有方法必须是抽象方法

2、接口中的变量声明为public static final，可以省略不写，但是默认为这个

3、接口中不能含有静态代码块以及静态方法，而抽象类可以有静态代码块和静态方法；

4、接口中的方法必须是public abstract类型的，可以省略不写，但是默认为这个

5、接口中的方法必须全部被子类实现

抽象类是对事务的抽象，接口是行为的抽象

25、jvm：类加载器将字节码装进内存的方法区，字节码验证器验证准确性，解释器执行文件

26、抽象类和接口的区别

a.一个类只能继承单个类，但是可以实现多个接口

b.抽象类中可以有构造方法，接口中不能有构造方法

c.抽象类中的所有方法并不一定要是抽象的，你可以选择在抽象类中实现一些基本的方法。而接口要求所有的方法都必须是抽象的

d.抽象类中可以包含静态方法，接口中不可以

e.抽象类中可以有普通成员变量，接口中不可以

27、线程，线程池：

1）run()方法用来运行线程，但只是作为主线程中的普通方法来调用，start()方法用来创建新的线程，会自动执行run()来达到多线程的目的

2）线程池：程序启动时，创建的若干线程来响应处理，因为创建线程需要时间，而且每个进程创建的线程数目有限，所以引入线程池的概念

3）在java中要想实现多线程的方法：继续Thread类，实现Runable接口，实现Callable接口。

28、垃圾回收机制：引用计数法，将没有被引用的内存地址进行回收

29、多态实现的必要条件：继承、方法重写、父类引用指向子类对象

30、集合：set，linklist，vector，arraylist，map

set：元素唯一，hashset底层使用哈希表，treeset底层使用二叉树（适用于元素排序）；

linklist：底层使用循环链表，查询慢，增删快，线程安全低

arraylist：底层使用数组，查询快，增删慢，线程安全低

vector：底层使用数组，查询快，增删慢，线程安全高

map：键值对，hashmap底层使用哈希表
